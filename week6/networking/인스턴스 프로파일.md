## 1. IAM 역할을 담는 인스턴스 프로파일 (Instance Profile) 
### 1) 개념 및 역할
* **권한 전달의 매개체:** EC2 인스턴스가 실행될 때 IAM 역할을 안전하게 전달하기 위한 컨테이너
* **구조적 필수성:** AWS 내부 아키텍처 상, EC2는 IAM 역할을 직접 인식하지 못하며 반드시 인스턴스 프로파일이라는 중간 단계를 거쳐야 함
* **자격 증명 자동화:** 인스턴스 내 어플리케이션이 AWS SDK/CLI를 사용할 때, 별도의 액세스 키 설정 없이 임시 보안 자격 증명을 자동으로 획득하게 해줌

### 2) 관리 방식
* **콘솔(Console):** IAM 역할을 생성할 때 AWS가 인스턴스 프로파일을 동일한 이름으로 자동 생성하므로 사용자가 개별적으로 관리할 필요가 없음
* **IaC/CLI (Terraform, CloudFormation):** IAM 역할과 인스턴스 프로파일을 각각 리소스로 정의하고 연결
