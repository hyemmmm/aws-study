# 📚 그림과 작동 원리로 이해하는 AWS 구조·서비스 스터디

본 저장소는 AWS의 핵심 서비스와 전체적인 구조를 학습하고, 주니어 개발자들의 발표 자료 및 실무 인사이트를 기록하는 공간입니다. 

단순한 이론 정리를 넘어, 실무와 개인 프로젝트에서 "왜 이렇게 설계해야 하는가?"에 대한 답을 찾는 것을 목표로 합니다.

---

## 🧑‍💻 스터디 개요
- **교재**: [그림과 작동 원리로 쉽게 이해하는 AWS 구조와 서비스](https://product.kyobobook.co.kr/detail/S000200882352)
- **대상**: 주니어 개발자 (백엔드, 프론트엔드, 클라우드 BD)
- **목표**: 
    - AWS 주요 서비스의 동작 원리 및 전체 흐름 이해
    - 실무/개인 프로젝트 적용을 위한 아키텍처 설계 기준 정립
    - 각 직군별 관점 공유를 통한 기술적 시야 확장

---

## ⚙️ 진행 방식
스터디는 **매주 일요일 오후 2시**에 진행되며, 회차당 2시간 동안 아래와 같이 구성됩니다.

1. **발표 및 질의응답 (1시간)**
   - 지정된 챕터의 발표자가 흐름을 주도하며 핵심 내용을 공유합니다.
   - 발표 자료는 스터디 전까지 본 저장소에 업로드합니다.
2. **경험 공유 및 네트워킹 (1시간)**
   - 해당 주차 주제와 연관된 실무 경험, 개인 프로젝트 적용 사례 등을 자유롭게 나눕니다.
   - 직접 구축하지 않았더라도 회사에서 운영 중인 인프라 구조에 대해 아는 만큼 공유하고 토론합니다.

## 📅 커리큘럼 및 발표 일정

| 회차 | 주제 | 범위 | 발표자 |
| :--- | :--- | :--- | :--- |
| **1회** | AWS 기초 및 시작 | 01~02장: 클라우드 용어 및 사용 시작 | 혜민 |
| **2회** | 컴퓨팅 서비스 | 03장: EC2, Lambda, 컨테이너(ECS/Fargate) | 하빈 |

| **4회** | 네트워크 서비스 | 05장: VPC, ELB, Route 53, CloudFront | 한나 |
| **5회** | 데이터베이스 | 06장: RDS, Aurora, DynamoDB | 연비 |
| **6회** | 보안 서비스 | 07장: IAM, CloudTrail, Config, WAF | 연비 |
| **7회** | 기타 및 관리 서비스 | 08장: 데이터 분석, ML, CloudWatch | 혜민 |
